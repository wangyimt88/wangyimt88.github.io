iOS进程之间的通讯

iOS中进程之间的通讯方式有很多种，我介绍一下2种最简单、最实用的方式。

第一种：只通讯，不需要传值

首先注册通知，一般放在程序启动的方法里面

CFNotificationCenterAddObserver(CFNotificationCenterGetDarwinNotifyCenter(),
                                    NULL,
                                    &callback,
                                    CFSTR("com.relaunchSpringBoard"),
                                    NULL,
                                    CFNotificationSuspensionBehaviorCoalesce);

com.relaunchSpringBoard 这个是自己定义的字符串

static void callback(CFNotificationCenterRef center,void *observer,CFStringRef name,const void     *object, CFDictionaryRef userInfo){
    NSString *strNotficationName=(__bridge NSString*)name;
    if ([strNotficationName isEqualToString:@"com.relaunchSpringBoard"]) {
        NSLog(@"--com.relaunchSpringBoard--");
    }
}

上面这个是回调方法，根据你推送的标识，就可以处理你的其他操作了

然后就是调用推送方法了，很简单，一句代码搞定：

#import <notify.h>

notify_post("com.relaunchSpringBoard");

第二种：既通讯又传值
首先把这个库和文件下载下来https://github.com/wangyimt88/rocketbootstrap直接拖到工程即可
然后注册通知：

CPDistributedMessagingCenter * messageCenter = ((id (*) (id, SEL, id))objc_msgSend)(objc_getClass("CPDistributedMessagingCenter"), @selector(centerNamed:), RocketBootstrap_xxx);
rocketbootstrap_distributedmessagingcenter_apply(messageCenter);

((void(*)(id, SEL))objc_msgSend)(messageCenter, @selector(runServerOnCurrentThread));
((void(*)(id, SEL, id, id, SEL))objc_msgSend)(messageCenter, @selector(registerForMessageName:target:selector:), @"myMessageName", self, @selector(handleMessage:withUserInfo:));

RocketBootstrap_xxx 这个是自己定义的宏定义
回调函数：

- (void)handleMessage:(id)arg1 withUserInfo:(id)arg2 {
    NSLog(@"arg2--%@", arg2);
}

返回的值就是上面方法的第二个参数arg2

接下来就是发送通知的方法了：

NSDictionary * userInfo = @{@"key1":value1,
                            @"key2":value2};
CPDistributedMessagingCenter * messageCenter = ((id (*) (id, SEL, id))objc_msgSend)(objc_getClass("CPDistributedMessagingCenter"), @selector(centerNamed:), RocketBootstrap_xxx);
rocketbootstrap_distributedmessagingcenter_apply(messageCenter);
((void(*)(id, SEL, id, id))objc_msgSend)(messageCenter, @selector(sendMessageName:userInfo:), @"myMessageName", userInfo);

这个通讯传值是非常方便的，但是有一个弊端，他不能在那种有沙盒的第三方应用注册通知，比如AppStore下载的应用，所以一般用在SpringBoard进程里注册通知。
所以第二种方法适用于Tweak与应用程序之间的通讯。
